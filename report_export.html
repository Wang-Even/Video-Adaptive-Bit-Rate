自适应码率 ABR 实验报告

一、实验目的

本实验围绕视频传输中的 Adaptive Bit-Rate（ABR）问题展开。实验目标是理解并实现基于缓冲区的 BBA 算法和基础的基于吞吐量预测的 RB 算法，在真实网络 trace 下比较二者在视频质量、卡顿时间、码率波动和 QoE reward 上的表现。同时，通过修改 BBA 的 reservoir、cushion 参数以及 reward 惩罚权重，分析不同参数对算法决策的影响。

三、算法实现

3.1 BBA 算法

BBA（Buffer-Based Algorithm）根据客户端当前缓冲区大小选择下一视频块的码率。代码中使用两个核心参数：

  • reservoir：低缓冲区阈值。当 buffer 小于该值时，选择最低码率，优先避免卡顿。
  • cushion：线性增长区间。当 buffer 位于 reservoir 和 reservoir + cushion 之间时，根据 buffer 大小线性映射到不同码率。

BBA 决策逻辑为：

if buffer_size < reservoir:
    bit_rate = 0
elif buffer_size >= reservoir + cushion:
    bit_rate = A_DIM - 1
else:
    bit_rate = (A_DIM - 1) * (buffer_size - reservoir) / cushion

缓冲区越高，BBA 越倾向于选择高码率；缓冲区越低，BBA 越保守。

3.2 RB 算法

RB（Rate-Based）算法主要根据历史吞吐量预测未来带宽。本实验实现的基础 RB 使用最近 5 个视频块的实测吞吐量均值作为带宽预测值，并乘以安全系数 0.9，避免过于激进。

吞吐量计算方式为：

throughput_kbps = video_chunk_size * 8 / download_time / 1000

之后从可选码率集合 [300, 750, 1200, 1850, 2850, 4300] 中选择不超过预测带宽的最高码率。

3.3 QoE Reward 公式

代码中的 reward 由三部分组成：

reward = video_quality - rebuf_penalty * rebuffer_time - smooth_penalty * bitrate_change

对应含义如下：

组成部分 含义 影响
视频质量奖励 当前视频块码率越高，奖励越高 鼓励选择高清晰度
卡顿惩罚 卡顿时间越长，惩罚越大 鼓励减少 rebuffer
平滑惩罚 相邻视频块码率差异越大，惩罚越大 鼓励码率稳定

默认参数为 rebuf_penalty=4.3、smooth_penalty=1。

六、BBA 参数影响分析

本实验比较 3 组 BBA 参数：

参数组 reservoir cushion 策略倾向
激进 2 8 更早升码率，更追求画质
默认 5 10 平衡画质与卡顿
保守 10 20 更晚升码率，更重视稳定

不同参数下的平均结果如下：

参数组 平均码率 Kbps 平均总卡顿时间 s 平均码率波动 Kbps 平均总 reward
BBA r=2,c=8 1874.49 17.626 610.07 -13.224
BBA r=5,c=10 1695.24 4.961 575.35 34.118
BBA r=10,c=20 1560.88 3.336 385.07 43.657

可以看出，较小的 reservoir 和 cushion 会让算法更激进，平均码率提高到 1874.49 Kbps，但平均卡顿时间增加到 17.626 秒，最终 reward 反而最低。较大的 reservoir 和 cushion 会让算法更保守，平均码率下降到 1560.88 Kbps，但卡顿和码率波动明显降低，平均 reward 最高。

因此，在移动网络波动较强的场景下，保守参数更适合 BBA；在带宽稳定且 buffer 能够维持较高水平时，较激进参数可以提升视频质量。

七、Reward 惩罚权重影响分析

本实验在 BBA 默认参数 reservoir=5,cushion=10 下修改 reward 权重：

设置 rebuf_penalty smooth_penalty 平均码率 Kbps 平均总卡顿时间 s 平均码率波动 Kbps 平均总 reward
默认 4.3 1.0 1695.24 4.961 575.35 34.118
增大卡顿惩罚 6.0 1.0 1695.24 4.961 575.35 25.684
增大平滑惩罚 4.3 2.0 1695.24 4.961 575.35 6.501

需要注意的是，在本实验代码中，reward 权重只用于计算 QoE 指标，不参与 BBA 决策。因此修改惩罚权重后，码率选择、卡顿时间和码率波动不变，但最终 reward 会发生变化。

当 rebuf_penalty 从 4.3 增加到 6.0 时，存在卡顿的 trace reward 降低，尤其是 norway_train_1 下降明显。当 smooth_penalty 从 1.0 增加到 2.0 时，BBA 因为码率波动较大，平均 reward 从 34.118 降到 6.501。这说明 reward 公式会显著影响对算法优劣的评价标准。

八、综合分析

BBA 的优势是实现简单，不需要显式预测网络带宽。当 buffer 较高时，BBA 可以大胆选择高码率，在平均带宽较高且波动较小的网络中能取得较高视频质量。其不足是对突发带宽下降不敏感，因为 BBA 只观察 buffer。当网络突然变差时，buffer 的下降具有滞后性，算法可能仍选择较高码率，从而导致卡顿。

RB 的优势是能够直接感知下载吞吐量变化，在带宽下降时更快降低码率，因此在 norway_car_1 和 norway_train_1 这类波动较强的 trace 上卡顿更少、码率更平稳。其不足是依赖历史吞吐量预测，若预测窗口过短会受瞬时波动影响，预测窗口过长又会响应滞后；同时安全系数会让 RB 偏保守，导致平均码率低于 BBA。

网络波动对 ABR 的影响主要体现在三个方面：

  1. 带宽突降会导致下载时间增加，如果 buffer 不足，就会产生卡顿。
  2. 带宽频繁波动会迫使算法频繁调整码率，增加质量波动惩罚。
  3. 在低平均带宽 trace 中，高码率策略虽然短期提高画质，但更容易因为卡顿惩罚导致总 reward 降低。

九、结论

本实验实现并比较了 BBA 与基础 RB 算法。实验结果表明，BBA 更适合平均带宽较高、buffer 能够保持充足的网络环境；RB 更适合带宽波动较大、突发下降明显的移动网络环境。

在 BBA 参数方面，较激进的 reservoir=2,cushion=8 虽然提高了平均码率，但明显增加卡顿，最终 QoE 较差；较保守的 reservoir=10,cushion=20 降低了码率和波动，在本实验选取的 3 条移动 trace 上取得了最高平均 reward。

在 reward 公式方面，视频质量、卡顿和码率波动之间存在明显权衡。卡顿惩罚越大，算法评价越偏向稳定播放；平滑惩罚越大，频繁升降码率的策略评价越低。因此，在实际 ABR 设计中，应根据业务需求合理设置 QoE 权重。
